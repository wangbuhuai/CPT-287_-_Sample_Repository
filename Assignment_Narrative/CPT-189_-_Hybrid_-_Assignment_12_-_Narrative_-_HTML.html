Program Overview

In this assignment, you are going to write a Java program to play a token-taking game.  You program should use recursion to find a "path" to reach exactly tokens within turns.

Your program must be recursive in this assignment, or you will receive zero credit.

Rules of the Game

1)At the beginning of the game, you have exactly 27 tokens.

2)In each turn of the game, you may do 1 of the following 2 things.

•Ask for exactly 39 more tokens.

•If the number of tokens you have is an even number, you may give back exactly half of the tokens you have.  However, if the number of tokens you have is an odd number, you do not have this choice and can only choose to ask for 39 more tokens.

3)The object of the game is to reach exactly tokens within turns, where (positive integer) and (positive integer) are specified at the beginning of the game.

4)The result of the game could be 1 of the following 2 results.

•The game reaches a dead end, which means that it is impossible to reach exactly tokens within turns.

•A "path" to reach exactly tokens within turns is found.  Please note that you do not need to use exactly turns.  is the maximum number of turns allowed in this game.

5)For simplicity, you do not need to do the following in your games.

•You do not need to find all the paths (sequences) from the starting state to the winning state.  Only outputting 1 valid path is sufficient.

•You do not need to count the number of valid sequences.

•You do not need to worry about whether the sequence you found is the shortest path or not.

The Game Class

1)The class has the following data fields.

•target.  This is a final integer representing the number of tokens to reach.  You must reach exactly this number of tokens to win the game.

•turns.  This is an integer representing the number of turns left.  This number will decrease by 1 after each move.  If the number has gone down to 0 but the current number of tokens is not equal to target, you lose the game.

•path.  This is an ArrayList of Characters that stores the path to reach the winning state.  If it is impossible to win the game, then the ArrayList will be empty.

2)The class has a constructor that takes two integers (representing the target number of tokens and the maximum number of turns allowed) as arguments.  The constructor initializes the target and turns with the values passed in, and initializes path with empty ArrayList.

3)The class has a method, play(), that takes an integer, tokens, as its only argument.  The method returns a boolean, where a true means that a valid path is found and false means that a dead end is reached.

•This method is recursive, which attempts to find a valid sequence to win the game.

•The first base case is that if tokens is equal to target, then you already win the game.  The method returns true immediately.

•The second base case is that if variable turns has dropped to 0, which means that you have used all the allowed turns and still not reached the target, then it is a dead end.  The method returns false immediately.

•Then, your method should try to solve the problem.  First, you need to try to ask 39 more tokens (denoted as option A).  Your method should append 'A' to path, decrease turns by 1, and check play(tokens + 39).  If it returns true, then it means that this choice can lead to winning the game, so your method should return true;  if it returns false, then it means that this choice cannot lead to winning the game, so you need to "recover" to the previous state, which means that you need to delete the 'A' from the rear end of path and increase turns by 1.

•Then, if tokens is an even number, you need to try to give back half of the tokens (denoted as option B).  Your method should append 'B' to path, decrease turns by 1, and check play(tokens / 2).  If it returns true, then it means that this choice can lead to winning the game, so your method should return true;  if it returns false, then it means that this choice cannot lead to winning the game, so you need to "recover" to the previous state, which means that you need to delete the 'B' from the rear end of path and increase turns by 1.

•Finally, if both choice did not work, then your method should return false.

4)The class has a method, printResult(), which takes a PrintWriter as its only argument and returns void.  The method first checks whether path is empty.  If it is empty, then write "Dead End" to the PrintWriter; it is not empty, then write all the characters in the ArrayList in order to generate the valid sequence.

The Input File

1)The input file is a plain text file with filename input.txt.

2)Each line in the input file contains exactly 2 integers, which are the target and turns of the game (in that order).

3)You cannot assume (or guess) the number of games stored in the input file.  In other words, no matter how many games are stored in the input file, your program should correctly play all of them.

4)There may be empty lines at the beginning, in the middle, and/or at the end of the input file.  Your program should smartly skip those empty lines.

5)Please refer to the sample input files to better understand the input file format.

The Output File

1)The output file is a plain text file with filename output.txt.

2)After playing each game in the input file, your program writes the game result to the output file (one result per line).

3)Please refer to the sample output files to better understand the output file format.

The main() Program

1)Parse each game stored in the input file and create a Game object.

2)Play the game by calling the .play() method.

3)Write the game result to the output file by calling the .printResult() method.
